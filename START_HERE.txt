Q: Can I run this on Windows?
A: No, it's designed for Linux. Uses Linux system calls and Unix
   permissions which don't exist on Windows.

Q: Why PyQt5 instead of tkinter?
A: PyQt5 is more professional, better for this project, and commonly
   used in real file managers.

Q: What if I get permission error?
A: That's expected! It demonstrates the kernel boundary. Check if you
   have write permission on parent directory: ls -la /path/to/parent

Q: How do I trace system calls?
A: Use strace:
   strace -e trace=stat,openat,getdents64 python3 main.py

Q: Where's the file content editing?
A: Phase 1 focuses on OS concepts. File editing would be Phase 2.

Q: Can I add more features?
A: Yes! See docs/PROJECT_SUMMARY.md for suggested Phase 2 features.
